# プロジェクト概要

## コンセプト

海外のIT・AIメディアを横断し、

**AIが日本語で翻訳・要約・重要度を整理するニュースリーダー**

を作りたい。

ユーザーは英語を読めなくても、毎日数分で海外の重要ニュースをキャッチアップできる。

---

# 解決したい課題

海外のIT・AIニュースは一次情報として価値が高い。

しかし、

- 英語で読むのが負担
- メディアごとに巡回する必要がある
- 重要な記事が分からない
- 情報量が多すぎる

という問題がある。

その結果、

「あとで読もう」

となり、結局読まなくなってしまう。

---

# サービス思想

このサービスは

「英語の記事をそのまま翻訳する」

サービスではない。

AIを活用して

- 読む価値がある記事を選び
- 日本語で理解しやすく要約し
- 必要であれば元記事へすぐ飛べる

という体験を提供する。

AIは記事を書くのではなく、

**海外ニュースを読むための編集者**

として振る舞う。

# AIの活用ポイント

## 1. タイトル翻訳

英語タイトルを自然な日本語へ翻訳する。

単なる機械翻訳ではなく、

意味が伝わるタイトルを生成する。

---

## 2. 記事要約

記事本文から

- 何が起きたか
- なぜ重要なのか
- 誰に関係する話なのか

を短く要約する。

全文翻訳ではなく、

数十秒で内容を理解できることを重視する。

---

## 3. 今日読むべき5本

AIが毎日

「今日読む価値が高い記事」

を5本選定する。

選定基準例

- 業界への影響
- 技術的重要度
- 新規性
- エンジニアへの関連性
- 話題性

ランキングではなく、

「編集長のおすすめ」

のような位置付けを目指す。

---

## 4. 記事タグ付け

AIが記事へタグを付与する。

例

- LLM
- AI Agent
- OpenAI
- Anthropic
- Python
- AWS
- Security
- Frontend
- Backend

将来的にはタグ単位で記事を閲覧できるようにしたい。

---

# やらないこと

MVPでは以下は不要。

- ユーザー認証
- AIチャット
- コメント
- SNS機能
- いいね
- フォロー
- パーソナライズ推薦
- 記事全文の翻訳表示

全ユーザー共通のニュース体験を提供する。


# 一番重要な価値

ユーザーが欲しいのは

「海外ニュースの全文」

ではない。

ユーザーが欲しいのは

**海外IT・AIニュースの重要ポイントを、日本語で短時間に理解できること**

である。

AIによって

「読む負担」を減らし、

「知る価値」を最大化するサービスを目指したい。
