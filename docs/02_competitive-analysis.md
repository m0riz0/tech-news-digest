# 競合分析

> 机上調査(2026-07時点の一般知識)ベース。実装前に主要サービスの最新状況を再確認することを推奨。

## 1. 競合カテゴリの整理

「海外ITニュースを日本語で短時間にキャッチアップする」というジョブに対する代替手段を、カテゴリで整理する。

### A. 国内テック系ニュースメディア(翻訳・編集あり)

例: ITmedia、CNET Japan、TechCrunch Japan(終了)、Gigazine、Publickey

- **強み**: プロの編集による高品質な日本語記事
- **弱み**: 翻訳・執筆に人手がかかるため**掲載までのタイムラグが大きく、カバー範囲が狭い**。一次情報のごく一部しか日本語化されない
- **本サービスとの違い**: AIによる自動処理で「速く・広く・浅く」カバーし、深掘りは元記事へ誘導する

### B. ソーシャル・キュレーション型

例: はてなブックマーク(テクノロジー)、Hacker News、Reddit、X(Twitter)

- **強み**: 集合知による話題選定。速報性が高い
- **弱み**: 英語記事はタイトルも本文も英語のまま。ノイズが多く、重要度判断はユーザー任せ。「読む」までの負担は解消されない
- **本サービスとの違い**: 日本語タイトル+要約で「開かなくても概要が分かる」体験を提供

### C. パーソナライズ系テックフィード

例: TechFeed、Menthas、Feedly(+AI要約機能)、SmartNews/Google News のテックカテゴリ

- **強み**: 興味に応じた記事配信。Feedly等はAI要約機能も持つ
- **弱み**: 海外記事の**日本語要約**を標準体験として提供するものは少ない。Feedly のAI機能は有料かつ英語中心。設定の手間がある
- **本サービスとの違い**: 設定不要・全員共通の「編集された紙面」体験。開いた瞬間に完成された日本語ダイジェストがある

### D. ニュースレター型

例: TLDR Newsletter、Import AI、Ben's Bites(いずれも英語)、国内では個人運営の翻訳ニュースレター

- **強み**: 「編集者が選んだ数本を短時間で」という体験は本サービスの思想に最も近い
- **弱み**: 英語のものが主流。日本語のものは人力運営で持続性・網羅性に課題。プッシュ型なので過去分の一覧性が低い
- **本サービスとの違い**: 同じ「編集者体験」をAIで自動化・日本語化し、Web上でいつでも閲覧できる

### E. AI要約・翻訳ツール(汎用)

例: ChatGPT/Claudeに記事URLを渡す、ブラウザの翻訳機能、要約系ブラウザ拡張

- **強み**: 任意の記事を処理できる
- **弱み**: **記事を見つけて、渡して、待つ**という能動的な操作が毎回必要。習慣化しない
- **本サービスとの違い**: 収集〜選定〜要約まで全自動。ユーザーは開くだけ

## 2. ポジショニングマップ

```
                日本語で読める
                     ▲
                     │
   国内テックメディア │  ★ 本サービス
   (人力・狭く深く)  │  (AI・広く速く)
                     │
 ◀───────────────────┼───────────────────▶
 編集済み(受動的)                キュレーションなし(能動的)
                     │
   TLDR等の英語NL     │  Hacker News / X
                     │  Feedly(素のRSS)
                     │
                     ▼
                 英語のまま
```

本サービスは「**日本語 × 編集済み(受動的) × 自動・即日**」の象限を狙う。この象限は人力では持続コストが高く、AIによる自動化で初めて成立する。

## 3. 差別化ポイントまとめ

| 観点 | 既存の代替手段 | 本サービス |
|---|---|---|
| 言語 | 英語のまま or 一部のみ日本語化 | 全記事を日本語タイトル+日本語要約 |
| 選定 | ユーザー任せ or 集合知 | AIが「今日読むべき5本」を毎日選定 |
| 速報性 | 人力翻訳はラグ大 | 収集から数時間以内に日本語化 |
| 網羅性 | メディア単位で分断 | 複数メディア横断 |
| 手間 | 巡回・設定・操作が必要 | 開くだけ(設定不要) |
| 一次情報 | 到達しにくい | 全記事に元記事リンク(誘導が前提の設計) |

## 4. リスク・脅威

| 脅威 | 内容 | 対応方針 |
|---|---|---|
| 大手の参入 | Google News / SmartNews / Feedly がAI日本語要約を標準搭載する可能性 | 「エンジニア向けIT・AI特化」の選定品質とタグ体系で差別化。ニッチ特化を維持 |
| 生成AIチャットの一般化 | 「ChatGPTに聞けばいい」層の拡大 | 能動操作不要の「紙面」体験と習慣化(毎朝開く場所になる)で対抗 |
| メディア側の対抗 | フィード縮小(全文→抜粋化)、AIクローラーのブロック | RSSの範囲で成立する設計を基本とし、本文取得は補助扱いに([09参照](./09_legal-operational-concerns.md)) |
| 同種個人サービス | 同じ発想のAIまとめサイトは技術的参入障壁が低い | 要約品質・選定品質のチューニングと継続運用が実質的な差別化要素 |

## 5. 結論

「海外ITニュースの日本語自動ダイジェスト」を全員共通の編集済み体験として提供するサービスは、現状明確な直接競合が確立していないニッチ。MVPで検証すべき仮説は:

1. AI要約の品質が「元記事を開かなくても概要が分かる」水準に達するか
2. 「今日の5本」の選定がユーザーの重要度感覚と一致するか
3. 毎日開きたくなる(習慣化する)体験になるか
