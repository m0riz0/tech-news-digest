# MVP整理

## 1. MVPの目的

前提となる位置付け(優先順):

1. **自分専用の運用テスト** — まず自分が毎日使い、品質と運用を検証する
2. **転職活動のポートフォリオ** — 設計書・アーキテクチャ・実装品質そのものをアピール材料にする
3. マネタイズは将来の別判断 — そのため**ランニングコストを極力ゼロに抑える**(AI処理は Pro サブスク枠、インフラは無料枠)

### 検証したい仮説

| # | 仮説 | 検証方法 |
|---|---|---|
| H1 | AI要約は「元記事を開かなくても概要が分かる」品質になる | 自分で日次利用し、要約の正確性・有用性を確認 |
| H2 | 「今日の5本」の選定は重要度感覚と一致する | 同上。選定理由の納得感を確認 |
| H3 | 毎日開きたくなる体験になる | 継続利用されるか(まず自分が毎朝使うか) |
| H4 | Pro プランのレート枠内で安定運用できる(普段の Claude Code 利用を圧迫しない) | `batch_runs` の実行記録とレート制限エラーの頻度を確認 |

## 2. MVPスコープ

### 2.1 In Scope(実装する)

**バッチ(裏側)**

- RSS/Atomフィード収集(初期 5〜10 メディア、DB設定で追加可能)
- 記事の重複排除・保存
- AIダイジェスト処理: 日本語タイトル + 日本語要約 + タグ付け(1記事1API呼び出し)
- 「今日読むべき5本」の日次選定(選定理由付き)
- 処理ログ・コスト記録

**Web(表側)**

- トップ画面: 今日の5本 + 最新記事一覧(単一ページ)
- 記事カード: メディア名 / 日本語タイトル / AI要約 / 公開日時 / タグ / 元記事リンク
- 過去記事のページネーション(または「もっと見る」)
- レスポンシブ対応

### 2.2 Out of Scope(やらない — BRIEFで明示されたもの)

| 機能 | やらない理由 |
|---|---|
| ユーザー認証 | 全員共通体験のため不要。実装コスト大 |
| AIチャット | コンセプト外(編集者であって対話相手ではない) |
| コメント / SNS / いいね / フォロー | コミュニティ機能はMVP検証に不要 |
| パーソナライズ推薦 | 全員共通の「紙面」がコンセプト |
| 記事全文の翻訳表示 | 著作権リスク大 + 「要約で十分、深掘りは元記事へ」の思想に反する |

### 2.3 Out of Scope(やらない — 設計判断で追加したもの)

| 機能 | 判断理由 |
|---|---|
| タグでの記事絞り込みUI | タグ付与自体は行う(データは貯める)が、閲覧UIは将来対応 |
| 検索 | 記事数が少ないMVP期は不要 |
| 日別アーカイブページ | daily_picks テーブルに日次データは残すため、後から追加可能 |
| 重複ニュースのグルーピング | AI処理が複雑化する。まず単純な1記事1カードで検証 |
| RSS以外の収集(スクレイピング前提のメディア) | 法的・技術的コストが高い。RSSを提供するメディアに限定 |

RSSフィードの存在と規約を実装時に確認の上、確定する。

| メディア | 種別 | 備考 |
|---|---|---|
| OpenAI Blog | 企業公式 | |

| Google AI Blog (blog.google/technology/ai) | 企業公式 | |
| GitHub Blog | 企業公式 | |

| Hugging Face Blog | 企業公式 | |
| TechCrunch | 商業メディア | カテゴリフィード(AI)を利用 |
| The Verge | 商業メディア | |
| Ars Technica | 商業メディア | |
| InfoQ / VentureBeat | 商業メディア | 初期はどちらか一方でも可 |

企業公式ブログ(自社告知)と商業メディア(報道)をバランスさせる。企業公式は権利リスクが低く速報性が高い。

## 5. 完了条件(Definition of Done)

- [ ] バッチが1日3回自動実行され、新着記事が日本語化されて表示される
- [ ] 毎朝「今日の5本」が自動更新される
- [ ] 障害時(フィード取得失敗・AI失敗)もサイト閲覧は継続できる
- [ ] ランニングコストが **AI追加費用 $0(Pro サブスク枠)+ インフラ無料枠内** に収まっている([04 §7参照](./04_architecture.md#7-コスト見積もり))
- [ ] バッチ実行が普段の Claude Code 利用のレート枠を圧迫していない
- [ ] About/免責ページがある

## 6. MVP後のロードマップ(BRIEFの将来機能の優先度案)

| フェーズ | 機能 | 前提 |
|---|---|---|
| v1.1 | タグ検索・タグ別一覧 | タグデータはMVPから蓄積済み |
| v1.1 | 日別アーカイブ | daily_picks データ蓄積済み |
| v1.2 | お気に入りメディア・既読管理(localStorage) | 認証不要のまま実現可能 |
| v1.3 | 週間まとめ(AI生成) | 記事データ蓄積後 |

| 判断次第 | **Anthropic API モードへの移行** | 収益化・公開拡大・レート枠逼迫のいずれかが発生したら。`AI_PROVIDER` 切替のみ([04 §8参照](./04_architecture.md#8-ai実行アダプタとapi移行パス)) |
